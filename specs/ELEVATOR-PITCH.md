# Grackle: Multi-Agent Orchestration for AI Coding Agents

Grackle is an open-source orchestration platform that turns AI coding agents into coordinated teams. Today, Claude Code, GitHub Copilot, and OpenAI Codex each run one-at-a-time in a single terminal on a single machine. Grackle changes that. Define a project, break it into a task DAG with dependencies, and Grackle spawns agents across Docker containers, SSH remotes, GitHub Codespaces, or your local machine — each on its own isolated git branch. Agents share discoveries through a structured findings system, work through dependency chains automatically, and surface their output through real-time streaming to a web dashboard or CLI. When an agent finishes a task, you review the diff, approve or reject with notes, and Grackle auto-unblocks the next tasks in the graph. The vision: you describe the architecture, Grackle coordinates the fleet.

Under the hood, Grackle is a Rush monorepo with five packages built on ConnectRPC and protobuf. A central gRPC server (SQLite, 36 typed RPCs with server streaming) manages state and bridges to a web UI via WebSocket. Pluggable environment adapters handle compute lifecycle — provisioning containers, opening SSH tunnels, bootstrapping remote nodes — while a companion gRPC service called PowerLine runs inside each environment and abstracts away the differences between agent SDKs through a common runtime interface. Credentials are encrypted at rest (AES-256-GCM), pushed to environments on demand, and never exposed through the API. Everything is observable: every tool call, every text block, every finding is streamed, persisted to JSONL, and rendered in a React dashboard with DAG visualization, diff viewing, and live event logs. Runtime-agnostic, environment-agnostic, fully typed, and designed for humans to stay in the loop.
